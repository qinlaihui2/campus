# CampusHub 服务器运维手册

> 服务器：阿里云 ECS `120.55.76.100` | 系统：Alibaba Cloud Linux 8

---

## 一、环境概览

| 服务 | 运行方式 | 端口 | 凭证 |
|------|---------|------|------|
| MySQL 8.0 | Docker (`mysql_twwr-mysql_TwWr-1`) | 13306 | root / root |
| Nginx | 主机（宝塔面板） | 80 | — |
| 后端 | 主机 Maven 运行 | 8080 | — |
| 前端 | Nginx 静态文件 | 80 | — |

## 二、数据库操作

```bash
# 连接数据库
docker exec mysql_twwr-mysql_TwWr-1 mysql -u root -proot campus_assistant

# 查看所有用户
docker exec mysql_twwr-mysql_TwWr-1 mysql -u root -proot -e "SELECT id, username, nickname, role, status FROM user;" campus_assistant

# 删除/重置 admin 账号（重启后端会自动重建）
docker exec mysql_twwr-mysql_TwWr-1 mysql -u root -proot -e "DELETE FROM user WHERE username='admin';" campus_assistant
```

## 三、后端启动

**重要**：必须指定 Java 21，Spring Boot 3.4.5 需要 Java 17+。

```bash
# 1. 切换到项目目录
cd /opt/claude-myblog/campus-server

# 2. 如果 admin 账号坏了，先删除再启动（DataInitializer 会自动创建 admin/admin123）
docker exec mysql_twwr-mysql_TwWr-1 mysql -u root -proot -e "DELETE FROM user WHERE username='admin';" campus_assistant

# 3. 拉取最新代码（如有改动）
cd /opt/claude-myblog && git pull

# 4. 指定 Java 21 启动后端
export JAVA_HOME=/usr/lib/jvm/java-21-openjdk
cd /opt/claude-myblog/campus-server
nohup mvn -pl campus-app spring-boot:run -Dspring-boot.run.profiles=server > /tmp/campus.log 2>&1 &

# 5. 查看启动日志（出现 "Started CampusApplication" 即成功）
tail -f /tmp/campus.log

# 6. 验证
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'
```

### 停止后端
```bash
ps aux | grep CampusApplication | grep -v grep | awk '{print $2}' | xargs kill
```

## 四、前端部署

前端通过 Nginx 提供静态文件，API 请求由 Nginx 反向代理到后端 8080 端口。

### 4.1 构建前端

```bash
# 服务器上构建（需先安装 Node.js：dnf install -y nodejs npm）
cd /opt/claude-myblog/campus-web

# 确认 .env.production 中 VITE_API_BASE_URL 为空（使用相对路径 /api）
cat .env.production
# 应显示：VITE_API_BASE_URL=

npm install
npm run build
```

### 4.2 部署到 Nginx

```bash
# 清空旧文件并复制新构建产物
rm -rf /usr/share/nginx/html/*
\cp -rf /opt/claude-myblog/campus-web/dist/* /usr/share/nginx/html/

# 重载 Nginx
nginx -s reload
```

### 4.3 Nginx 配置

宝塔面板管理 Nginx，主配置：`/www/server/nginx/conf/nginx.conf`
站点配置：`/www/server/panel/vhost/nginx/campus.conf`

```nginx
server {
    listen 80;
    server_name _;

    # 前端 SPA
    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API
    location /api/ {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 30s;
        proxy_read_timeout 60s;
    }

    # AI 对话 SSE 流式
    location /api/chat/send {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 600s;
        chunked_transfer_encoding on;
    }

    # WebSocket
    location /ws/ {
        proxy_pass http://localhost:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 3600s;
    }
}
```

配置文件修改后检查并重载：
```bash
nginx -t && nginx -s reload
```

## 五、一键完整启动

```bash
# ===== 1. 启动后端 =====

cd /opt/claude-myblog/campus-server

nginx -s reload
```

## 六、常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `密码错误` 但密码确定是对的 | admin 密码 hash 被手动改坏 | 删除 admin 后重启后端自动重建 |
| `npm: command not found` | 未装 Node.js | `dnf install -y nodejs npm` |
| Maven 报 `class file version 61.0` | JAVA_HOME 指向 Java 11 | `export JAVA_HOME=/usr/lib/jvm/java-21-openjdk` |
| 前端 404 | Nginx 配置或文件路径不对 | 检查 `/www/server/panel/vhost/nginx/campus.conf` |
| 前端 Network Error | API 地址写死为 `http://120.55.76.100:8080` | `.env.production` 中 `VITE_API_BASE_URL=` 留空 |
| MySQL 密码错误 | 非 Docker 直连 | 必须用 `docker exec mysql_twwr-mysql_TwWr-1 mysql ...` |
