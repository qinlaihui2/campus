```bash
cd E:\java\claude-myblog

```

## 服务器一键部署（前后端全更）

> **用 `git reset --hard` 避免日志文件冲突，每次部署都用这个**

```bash
cd /opt/claude-myblog && git reset --hard origin/master && cd campus-web && npm run build && rm -rf /usr/share/nginx/html/assets/* && \cp -rf dist/* /usr/share/nginx/html/ && cd ../campus-server && mvn clean install -pl campus-announcement,campus-market,campus-app -am -DskipTests -q && export JAVA_HOME=/usr/lib/jvm/java-21-openjdk && kill $(ps aux | grep CampusApplication | grep -v grep | awk '{print $2}') 2>/dev/null && nohup mvn -pl campus-app spring-boot:run -Dspring-boot.run.profiles=server > /tmp/campus.log 2>&1 &
```

> **注意**：只改了前端就跳过 `mvn clean install` 那一段；只改了后端某个模块就把 `-pl campus-xx` 换成对应模块名。

## 只更前端（不重启后端）

```bash
cd /opt/claude-myblog && git reset --hard origin/master && cd campus-web && npm run build && rm -rf /usr/share/nginx/html/assets/* && \cp -rf dist/* /usr/share/nginx/html/
```

## 只更后端某个模块

```bash
export JAVA_HOME=/usr/lib/jvm/java-21-openjdk
cd /opt/claude-myblog/campus-server && mvn clean install -pl 模块名,campus-app -am -DskipTests -q
kill $(ps aux | grep CampusApplication | grep -v grep | awk '{print $2}') 2>/dev/null
nohup mvn -pl campus-app spring-boot:run -Dspring-boot.run.profiles=server > /tmp/campus.log 2>&1 &
```

模块名对照：`campus-announcement` `campus-market` `campus-course` `campus-message` `campus-chat` `campus-square` `campus-user` `campus-security` `campus-feedback` `campus-file` `campus-knowledge` `campus-rag` `campus-my` `campus-lost-found` `campus-common` `campus-app`

## 查看后端状态

```bash
# 是否启动成功
grep -a "Started Campus" /tmp/campus.log

# 实时日志
tail -f /tmp/campus.log

# 只看错误
grep -a -i "ERROR\|Exception" /tmp/campus.log | tail -20

# 进程是否活着
ps aux | grep CampusApplication | grep -v grep
```

## 测试 API

```bash
# 登录
TOKEN=$(curl -s -X POST http://localhost:8080/api/auth/login -H "Content-Type: application/json" -d '{"username":"admin","password":"admin123"}' | grep -o '"accessToken":"[^"]*"' | cut -d'"' -f4)

# 查公告
curl -s http://localhost:8080/api/announcements?page=1 -H "Authorization: Bearer $TOKEN"

# 删公告
curl -s -X DELETE http://localhost:8080/api/admin/announcements/1 -H "Authorization: Bearer $TOKEN"

# 创公告
curl -s -X POST http://localhost:8080/api/admin/announcements -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" -d '{"title":"test","content":"hello","category":"NOTICE","priority":0}'
```

## 常见问题速查

| 现象 | 原因 | 解决 |
|------|------|------|
| git pull 冲突 `application.json` | 日志文件本地改了 | `git reset --hard origin/master` |
| 后端 404/老接口还在 | Maven 没重编子模块 | `mvn clean install -pl 模块名,campus-app -am` |
| `release version 21 not supported` | JAVA_HOME 指向 Java 11 | `export JAVA_HOME=/usr/lib/jvm/java-21-openjdk` |
| 前端改了不生效 | 浏览器缓存 | Ctrl+Shift+R 强制刷新 |
| Nginx 重启 | — | `nginx -s reload` |
| SSH 老断开 | 没设保活 | `ssh -o ServerAliveInterval=60 root@120.55.76.100` |
| 公告管理不显示新数据 | Redis 缓存未清 | Redis 清缓存命令 |
| 创建公告 500 | 已在代码中修复（cover_image 有默认值了） | — |
